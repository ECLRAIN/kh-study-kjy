## 진행 계획

- 매 주 프로젝트 관련된 시간을 수업 시간 내에 할당
  - 1회(2시간)의 회의
  - 1회(2시간)의 프로젝트 수업
- 6시간 수업이 있는 날인 경우 1시간으로 축소할 수 있음

## 진행 순서

1. 조장 선출
2. 프로젝트 주제 선정
3. 롤모델 사이트 선정
4. 요구사항 설계(+기능 산정)
5. 데이터베이스 관계도(ERD) 설계
6. UI 설계
7. Flow 설계
8. Github 팀 프로젝트 설정
9. 프로젝트 구현
10. 발표자료 제작 및 발표
11. 포트폴리오 작성
12. 애플리케이션 배포

## 홈페이지 종류

- 예매를 기반으로 한 홈페이지
- 전자상거래 홈페이지
- 그룹웨어
  - 인사/회계
  - 구매/재고관리
  - 기타(메신저, 프로젝트관리)
- 지리정보기반 홈페이지
- 금융(증권/보험/은행)용 홈페이지
- 의료기관용 홈페이지
- 공공기관용 홈페이지
